Current UI/UX Audit

Issues found in the Stone Harp Analytics hosted app — click the ones you want to prioritize

1

Flash / Error Glimpse on Load

Root cause: The splash screen calls st.set_page_config(), then time.sleep(3), then st.rerun(). During the rerun, the page config switches (centered → wide or vice versa), and Streamlit briefly shows unstyled content or layout shift before the login page renders. The 2-cycle cookie check in auth_ui.py also causes an extra flash — cycle 1 shows a loading screen, cycle 2 checks cookies, potentially cycle 3 shows login.

Fix: Eliminate layout config switching, smoother splash→login transition

2

Login Page — Too Simple, Centered Card

Current: Small centered card with logo + form on dark background. Looks functional but generic.

Proposed: Split-screen layout — left half: branding showcase (logo, tagline, decorative elements), right half: sign-in form. Premium feel, like modern SaaS login pages.

This is the main visual upgrade you requested

3

No Smooth Page Transitions

Current: CSS pageIn animation exists (fade + slide up) but gets overridden by Streamlit's rerun behavior. The JS page-loader script tries to intercept clicks but it's fighting Streamlit's framework. Result: pages just "snap" in with a brief white flash.

Fix: Streamlit fundamentally re-executes the script on navigation. True cross-fade is impossible, but we can: dim on exit, fade-in on entry, use the gold shimmer bar consistently, and prevent the content "jump" that looks jarring.

Improve perceived smoothness within Streamlit's constraints

4

Landing Page — 3 Cards Need Rebalancing

Issue: We just added Daedalus as the 3rd tool. The current 2-card layout was stretched to 3. The columns may feel cramped and the card descriptions are short. The "Sign Out" and "Admin Panel" buttons sit awkwardly below.

Rebalance the 3-card grid, integrate admin/logout more naturally

5

Security Review

Good: bcrypt hashing, rate limiting (5 attempts/5-min lockout), session timeout (30 min), remember-me with hashed tokens, activity logging.

Concerns:

  • Rate limiting is in-memory (_login_attempts dict) — resets on server restart, not persistent
  • Remember-me cookie is not HttpOnly (streamlit-cookies-controller uses JS cookies)
  • No CSRF protection on forms (Streamlit limitation)
  • Session ID not rotated on privilege changes
  • tool_access only supports single tool or "both" — no multi-tool combos

Most are Streamlit framework limitations, not code bugs

6

Sidebar Polish

Material icon text leaks are mostly handled by CSS, but the sidebar could use: better active-page highlighting, smoother hover states, and a more polished user info section at the bottom.

Minor polish, low priority
